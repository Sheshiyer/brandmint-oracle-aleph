# UX Principles for Premium Non-Technical Front UI

## Design intent
Create a calm, premium, and trustworthy experience that feels like a guided studio rather than an engineering console.

## Principles
1. **Product-MD-first entry**
   - First action is uploading or pasting Product MD.
   - Technical controls remain secondary and contextual.

2. **Narrative before configuration**
   - Show extracted understanding before asking for structured edits.
   - Explain “why this matters” before each step.

3. **Plain-language defaults**
   - Replace internal terminology with user-facing language.
   - Every complex action gets a one-line explanation and expected outcome.

4. **Calm confidence over urgency**
   - Soft visual hierarchy, clear progress, minimal alarm states.
   - Errors are framed as recoverable with explicit next actions.

5. **Editorial premium aesthetic**
   - Inspired by X references: clean grids, rich typography, controlled accents, tactile surfaces.
   - Avoid visual clutter and avoid terminal-like density.

6. **Progressive disclosure**
   - Start with essential choices only.
   - Advanced controls remain available without overwhelming first-time users.

7. **Trust and traceability**
   - Every generated result links back to source inputs and rationale.
   - Users can review, edit, and re-run with confidence.
